# Enterprise Hybrid Identity & IAM Lab

> Designing, deploying, and securing a modern enterprise hybrid identity environment using Active Directory, Microsoft Entra ID, Azure, Microsoft Sentinel, and enterprise IAM best practices.

---

# Project Overview

This project simulates the design, implementation, operation, and security monitoring of a modern enterprise hybrid identity environment.

The lab combines an on-premises Active Directory infrastructure with Microsoft Entra ID and Azure services to mirror how many organizations manage identity today.

The project emphasizes enterprise architecture, identity governance, role-based access control (RBAC), Conditional Access, Privileged Identity Management (PIM), Microsoft Sentinel, PowerShell automation, and Zero Trust security principles.

---

# Business Scenario

A fictional mid-sized enterprise is modernizing its identity infrastructure.

The organization currently relies on traditional on-premises Active Directory but is migrating toward a hybrid identity architecture to improve:

- Identity security
- Access governance
- Cloud integration
- Centralized authentication
- Privileged access management
- Security monitoring
- Operational automation

The goal is to design an environment that closely resembles a real enterprise deployment.

# Core Objectives

- Design a realistic enterprise identity architecture
- Deploy and configure Active Directory Domain Services
- Build a hybrid identity environment
- Synchronize identities with Microsoft Entra ID
- Implement Role-Based Access Control (RBAC)
- Configure Multi-Factor Authentication (MFA)
- Deploy Conditional Access policies
- Secure privileged accounts using PIM
- Automate user provisioning and offboarding with PowerShell
- Monitor identity activity using Microsoft Sentinel
- Create detection and incident response documentation
- Map security controls to Zero Trust and MITRE ATT&CK
